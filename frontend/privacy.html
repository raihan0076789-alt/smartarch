Section 01

Overview

SmartArch("we", "us", or "our") is an AI-powered architectural design and project management platform. We are committed to protecting the personal information of all users who register and use our services.

This Privacy Policy applies to all services provided through the SmartArch web application, including our main platform, AI design generation service, admin dashboard, and any associated APIs. By creating an account or using the platform, you acknowledge that you have read, understood, and agree to the terms described in this policy.

Plain English Summary: We collect only the data needed to run the platform — your account details, your design projects, and basic usage information. We do not sell your data. We do not serve ads. Your architectural designs belong to you.
Section 02

Data We Collect

We collect different categories of data depending on how you interact with the platform. All collection is limited to what is necessary to provide our services.

Account Information
Full name, email address, password (hashed), company name (optional), phone number (optional), and user role.
Project Data
Project names, descriptions, floor plans, room configurations, dimensions, materials, specifications, tags, and version history.
Preferences

Authentication Tokens

We do not collect payment card information (no payments are processed through the platform), government identification numbers, biometric data, or any sensitive personal information beyond what is listed above.

Section 03

How Your Data Is Used

All data we collect is used exclusively to operate and improve the SmartArch platform. Specifically:

  • Account Authentication: Your email and hashed password are used to verify your identity and grant secure access to your account and projects.
  • Project Storage & Retrieval: Project data (floor plans, rooms, materials, metadata) is stored in MongoDB and retrieved on demand to power your dashboard and design workspace.
  • AI Design Generation: Design parameters you provide (building type, dimensions, room requirements) are passed to our local AI service to generate floor plan layouts. These inputs are not retained by the AI model beyond the duration of the generation request.
  • Automated Cost Estimation: Your project's dimensions and floor count are used to automatically compute total area, estimated construction cost, and projected build time displayed in your project metadata.
  • Admin Analytics: Aggregated, non-personally-identifiable usage statistics (total user count, projects per day, status distributions) are displayed in the admin dashboard for platform monitoring.
  • Collaboration Features: Your name and email are used to identify you as a collaborator on shared projects you have been invited to join.
  • Platform Improvement: Usage patterns may be reviewed in aggregate form to identify bugs, improve performance, and prioritize new features.
We never use your data for advertising. SmartArch contains no advertising systems, no tracking pixels, and no third-party marketing integrations. Your project data is never used to train AI models operated by third parties.
Section 04

Authentication & Account Security

We take authentication security seriously. Here is exactly how your credentials are handled:

  • Password Hashing: Your password is never stored in plain text. When you register or update your password, it is hashed usingbcryptjswith a salt factor of 10 before being written to the database. The original password is never recoverable from our systems.
  • JWT Tokens: Upon successful login, a signed JSON Web Token (JWT) is issued containing your user ID and role. This token is stored in your browser's localStorage and included in the Authorization header of every API request. Tokens expire according to the system's configured expiry period.
  • Role-Based Access Control: The platform enforces three access levels—user,architect, and admin. Middleware on every protected route validates your token and role before granting access.
  • Admin Authentication: Admin accounts use a separate login endpoint that issues tokens with an additional isAdmin flag, verified by dedicated admin middleware on all admin routes.
  • Password Reset Tokens: If a password reset is initiated, a time-limited, cryptographically random token is generated and stored. This token is invalidated upon use or expiry.
Security recommendation: Use a strong, unique password for your SmartArch account. We recommend at least 12 characters including a mix of letters, numbers, and symbols. Never share your login credentials with anyone.
Section 05

Data Storage & Security

All user and project data is stored in a MongoDB database managed through the Mongoose ODM. The database structure is defined by validated schemas that enforce data types, field lengths, and required fields to maintain data integrity.

We implement the following security practices to protect stored data:

  • Passwords are never stored in recoverable form — only bcrypt hashes are persisted
  • Database credentials and JWT secrets are stored in environment variables (.env files), never in source code
  • All API routes that handle personal or project data require a valid, non-expired JWT token
  • Admin-only routes require both JWT validation and an admin role check via dedicated middleware
  • CORS policies restrict which origins can make API requests to the backend service
  • Request body size limits (10MB) prevent denial-of-service via oversized payloads
  • Sensitive password fields are excluded from all API responses using Mongoose's select: false option

While we implement industry-standard security practices, no system is 100% immune to security threats. We encourage users to use strong passwords and to notify us immediately if they suspect unauthorized access to their account.

Section 06

AI Services & External APIs

SmartArch uses an AI service to generate architectural floor plans based on parameters you provide. Here is how this works and what it means for your privacy:

Local AI Processing: Our AI service runs the Ollama phi3:mini model locally on our server infrastructure. Your design parameters are sent to this local service — they are not transmitted to any external third-party AI provider such as OpenAI, Anthropic, or Google. All AI processing occurs within our own controlled environment.
  • AI Input Data: When you request a floor plan, parameters including building type, floor count, total area, and desired room types are sent to the AI backend service (Port 3001). These parameters are used solely to generate your layout.
  • Image Uploads: If you upload a floor plan image for analysis, that image is processed locally by our edge detection system to extract room outlines. Images are processed in memory and are not permanently stored on our servers.
  • AI Output: Generated floor plan data (room positions, dimensions, layouts) is returned to your browser and can be saved as a project at your discretion. Unsaved AI outputs are not retained.
  • No Model Training: Your inputs to the AI service are never used to train, fine-tune, or update any AI model. The phi3:mini model operates in inference-only mode.

The platform's front-end communicates with two backend services: the Main API (Port 5000) for authentication, project management, and user operations, and the AI Service (Port 3001) for layout generation and image analysis. Both services are operated by SmartArch and subject to this privacy policy.

Section 07

Data Sharing & Third Parties

We do not sell, rent, or trade your personal information to any third parties. Your data is shared in only the following limited circumstances:

  • Project Collaborators: If you invite another user to collaborate on a project, your name and email will be visible to that collaborator within the project interface. You control who is invited.
  • Public Projects: If you mark a project as "public" (isPublic: true), its details may be visible to other authenticated users of the platform. You can revert a project to private at any time.
  • Platform Administrators: Authorized administrators with valid admin credentials can view user account information and project listings through the admin dashboard for the purpose of platform management, support, and moderation.
  • Legal Requirements: We may disclose information if required to do so by law, court order, or to protect the safety and rights of our users or the platform.

Beyond the above, your data remains within the SmartArch platform and is not shared with any external third-party companies, advertisers, analytics providers, or data brokers.

Section 09

Cookies & Browser Storage

SmartArch uses browser localStorage (not cookies) to store your session token and cached user profile on your device. Specifically:

  • token — Your JWT authentication token, used to authenticate API requests during your session. This remains until you log out or manually clear browser storage.
  • user — A JSON object containing your basic profile information (name, email, role) used to populate the UI without requiring a server request on every page load.
  • adminToken / adminUser — Equivalent tokens stored when an admin logs in through the admin panel. Cleared on admin logout.

We do not use third-party tracking cookies, analytics cookies (Google Analytics, etc.), or advertising cookies of any kind. You can clear your browser's localStorage at any time through your browser's developer tools or privacy settings, which will log you out of the platform.
